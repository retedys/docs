---
title: "Policies API"
description: "Manage company-scoped policies, versions, and publishing in the DocsOrb API."
---

Policies are company-scoped resources. Policy responses merge base policy fields with fields from the current or requested version where noted.

> **Important:** Policy deletion is soft-delete behavior. Version publishing is a separate action from draft editing.

## List policies

`GET /policies`

Required scope: `policies.read`

Returns policies for the calling company. The response merges base policy fields with current version fields.

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `board_id` | string | No | Filter by board ID |
| `status` | string | No | Filter by current version status |

### Example request

```bash
curl -X GET "https://api.docsorb.com/api/v1/policies?status=draft" \
  -H "Authorization: Bearer <api_key>"
```

## Create a policy

`POST /policies`

Required scope: `policies.write`

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Policy name |
| `board_id` | string | No | Board ID |
| `description` | string | No | Policy description |
| `category` | string | No | `security`, `privacy`, `hr`, `esg`, or `other` |
| `risk_level` | string | No | `low`, `medium`, or `high` |
| `tags` | string[] | No | Policy tags |
| `owner_id` | string | No | Org member ID |
| `version` | string | No | Defaults to `1.0` |
| `status` | string | No | Defaults to `draft` |
| `effective_from` | string | No | Effective date/time |
| `primary_language` | string | No | Primary language code |
| `framework_links` | JSON | No | Framework mapping data |
| `ai_overview` | string | No | Summary text |
| `ai_key_points` | JSON array | No | Key points |
| `content_html` | string | No | HTML content |
| `content_json` | JSON | No | Structured content |
| `stakeholder_ids` | string[] | No | Full stakeholder member ID array |
| `audience_group_ids` | string[] | No | Full audience group ID array |

Returns `201 Created`.

### Example request

```bash
curl -X POST "https://api.docsorb.com/api/v1/policies" \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Employee Handbook",
    "board_id": "board_8a58d7aa-9807-4ad7-a403-e3ef4e2ce4fb",
    "description": "Operational expectations and employee guidance",
    "category": "hr",
    "risk_level": "medium",
    "tags": ["onboarding", "people-ops"],

    "status": "draft",
    "primary_language": "en",
    "ai_overview": "Core handbook for employee onboarding and daily operations.",

    "audience_group_ids": ["group_59c93452-4e60-4113-bd03-f2bce9e82542"]
```

Required scope: `policies.read`

### Path parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `policyId` | string | Yes | Policy ID |

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `version` | string | No | Version label such as `1.0` |

### Supported fields

| Field |
|-------|
| `name` |
| `description` |
| `category` |
| `risk_level` |
| `tags` |
| `owner_id` |
| `board_id` |
| `effective_from` |
| `primary_language` |
| `framework_links` |
| `ai_overview` |
| `ai_key_points` |
| `content_html` |
| `content_json` |
| `status` |
| `stakeholder_ids` |
| `audience_group_ids` |

### Example request

```bash
curl -X PATCH "https://api.docsorb.com/api/v1/policies/policy_97d0f0ea-bca9-429f-9ca8-bb629c3dff16" \
```

## Delete a policy

`DELETE /policies/{policyId}`

Required scope: `policies.write`

Soft-deletes the policy and returns a success payload.

### Example request

```bash
curl -X DELETE "https://api.docsorb.com/api/v1/policies/policy_97d0f0ea-bca9-429f-9ca8-bb629c3dff16" \
  -H "Authorization: Bearer <api_key>"
```

### Example response

```json
{
  "success": true
}
```

## Create a policy version

`POST /policies/{policyId}/versions`

Required scope: `policies.write`

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | `minor` or `major` |

### Behavior

- `minor` copies the current version content forward.
- `major` creates a fresh draft version.

### Example request

```bash
curl -X POST "https://api.docsorb.com/api/v1/policies/policy_97d0f0ea-bca9-429f-9ca8-bb629c3dff16/versions" \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "minor"
  }'
```

### Example response

```json
{
  "version": {
    "id": "version_1ccf6bb7-6c90-4f09-b1ce-2792ba979f32",
    "version": "1.1",
    "status": "draft",
    "effective_from": null
  }
}
```

## Update a policy version

`PATCH /policies/{policyId}/versions/{version}`

Required scope: `policies.write`

Only draft versions can be edited directly.

### Supported fields

| Field |
|-------|
| `effective_from` |
| `primary_language` |
| `framework_links` |
| `ai_overview` |
| `ai_key_points` |
| `content_html` |
| `content_json` |
| `status` |
| `version` |

### Example request

```bash
curl -X PATCH "https://api.docsorb.com/api/v1/policies/policy_97d0f0ea-bca9-429f-9ca8-bb629c3dff16/versions/1.1" \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "ai_overview": "Updated handbook summary for v1.1",
    "status": "draft"
  }'
```

### Example response

```json
{
  "version": {
    "id": "version_1ccf6bb7-6c90-4f09-b1ce-2792ba979f32",
    "version": "1.1",
    "status": "draft",
    "ai_overview": "Updated handbook summary for v1.1"
  }
}
```

## Delete a policy version

`DELETE /policies/{policyId}/versions/{version}`

Required scope: `policies.write`

### Behavior

- DocsOrb cannot delete the only remaining version.
- If the deleted version is current, DocsOrb points the policy to the latest remaining version.

### Example request

```bash
curl -X DELETE "https://api.docsorb.com/api/v1/policies/policy_97d0f0ea-bca9-429f-9ca8-bb629c3dff16/versions/1.1" \
  -H "Authorization: Bearer <api_key>"
```

### Example response

```json
{
  "success": true
}
```

## Publish a policy version

`POST /policies/{policyId}/versions/{version}/publish`

Required scope: `policies.publish`

Marks the target version as approved and sets it as the policy's current version.

### Example request

```bash
curl -X POST "https://api.docsorb.com/api/v1/policies/policy_97d0f0ea-bca9-429f-9ca8-bb629c3dff16/versions/1.1/publish" \
  -H "Authorization: Bearer <api_key>"
```

### Example response

```json
{
  "version": {
    "id": "version_1ccf6bb7-6c90-4f09-b1ce-2792ba979f32",
    "version": "1.1",
    "status": "approved",
    "effective_from": "2026-05-01T00:00:00.000Z"
  }
}
```
